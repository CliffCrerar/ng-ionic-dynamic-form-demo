Whats this?

This is a dev experiment to understand the complexities around dynamic data driven form construction. while using Angular with Ionic components

Assumptions

It is possible to write an algorithm that consumes an API that produces a DTO stipulating specific enumeration of different types form controls. Consumption of this API also determines the configuration and content that send signals to the user, what actions are required of what actions is required.

Execution

Create a reusable generic Angular component to display a dynamic form. Tranlate the DTO into a model and use reflection, iteration and switching to enable the dynamic nature of the component.

Figure A Figure A Brief

Blue C and purple X is the form factory and contain the form control prototypes. Red R is the router module, Orange C and its views are the form containers. Orange and Purple Models are the that are respectively called upon that define a specific enumeration and configuration of a form.

The user selects a path to visit leading to on of the orange C controller components. This triggers a chain effect that passes a variable to blue C, this variable value is dependant on the route. The variable instructs the API on which DTO to call, the response is reflected upon in the view. The each iteration a form control n a list of form controls is switched on according the params on the DTO. once iteration completed the form is rendered.
